## 评分标准

- 5 分：优秀，无明显缺陷
- 4 分：较强，有小问题但不影响核心使用
- 3 分：合格，有明显问题需要改进
- 2 分：较弱，存在严重影响使用的问题
- 1 分：严重缺失或基本失效

## 安全维度评分专项说明

安全维度权重已从 5 提升为 10，反映其实际重要性。打分时需分三层评估：

| 护栏层 | 评分贡献 | 判断标准 |
|--------|----------|----------|
| 输入层（验证参数、禁止敏感数据） | 约 1.5 分贡献 | 有明确输入验证规则且禁止敏感参数传入 |
| 动作前层（执行前确认、最小权限） | 约 2 分贡献 | 危险操作有确认机制，不可逆操作要求显式确认 |
| 输出层（副作用验证、回滚路径） | 约 1.5 分贡献 | 有输出验证和失败恢复路径 |

**特别规则：**
- 仅有单层或笼统安全警告：最高 2 分
- 三层均有但不完整：3 分
- 三层完整但细节不足：4 分
- 三层完整且细节充分：5 分
- 高风险 skill（涉及密钥/生产/用户数据）若三层护栏不完整：强制标注为 P0 风险候选项

## Skill 类型修正说明

以下类型 skill 在特定维度上有额外的质量期待，评分时应参考：

| Skill 类型 | 额外关注点 |
|------------|-----------|
| 线性流程型 | 执行性中需有安全默认值和降级方案；每步应有具体命令示例 |
| 决策树型 | 结构质量中需评估渐进式披露是否合理；导航分支是否清晰 |
| 循环迭代型 | 验证与完成判定中需有退出检查清单；执行性中需有防偷懒机制 |
| 接力棒型 | 结构质量中需评估状态持久化机制；续命步骤是否有 MUST 标注 |
| 多阶段编排型 | 边界控制需评估子 skill 职责分离；验证需评估检查点质量 |
| 思维框架型 | 执行性中需有量化阈值（"最少N个"）；边界控制需有明确非目标约束 |
| 自主式（执行模式） | 验证维度要求机器可验证；安全维度需有超时边界；单独触发 Phase 5C 检查 |

## P0 阻断条件

满足以下**任一**条件即为 P0 阻断，不可上线：

1. 触发严重歧义（description 无法区分该用还是不用）
2. 主体内容不可执行（没有具体步骤、没有输入输出）
3. 没有结束条件或验证条件
4. 高风险动作（删除/写入/外部调用/密钥/生产环境）无任何 guardrail，或仅有单点笼统警告
5. scope 本质失控（混入多个不相关独立职责）
6. **（新）声明为自主式执行，但完成条件核心依赖用户实时干预**
7. **（新）高风险 skill（密钥/生产/用户数据）安全防护仅停留于单层，未覆盖输入/动作前/输出三层**
